How To Secure Apache with Let's Encrypt

SSL certificates are used within web servers to encrypt the traffic between the server and client, providing extra security for users accessing your application. Let’s Encrypt provides an easy way to obtain and install trusted certificates for free.

Prerequisites:

  • An Ubuntu server with a non-root sudo user, which you can set up by following our Initial Server Setup guide
  • A functional Apache web server installation hosting multiple virtual hosts
  • Step 1 — Install the Let's Encrypt Client
  • Step 2 — Set Up the SSL Certificate
  • Step 3 — Verifying Certbot Auto-Renewal

When you are ready to move on, log into your server using your sudo-enabled account.

If you like to learn from video tutorials I would suggest to watch https://www.youtube.com/.

For the purpose of this guide, we will install Let’s Encrypt certificates for the domains and sud-domains databucketsolutions.com and dashboard.databucketsolutions.com. These will be referenced throughout the guide, but you should substitute them with your own domains while following along.

Step 1 — Install the Let's Encrypt Client

Let's Encrypt certificates are fetched via client software running on your server. The official client is called Certbot, and its developers maintain their own Ubuntu software repository with up-to-date versions.

Because Certbot is in such active development it's worth using this repository to install a newer version than Ubuntu provides by default.

ajay@DataBucket:~$ sudo add-apt-repository ppa:certbot/certbot

You'll need to press ENTER to accept. Afterwards, update the package list to pick up the new repository's package information:

ajay@DataBucket:~$ sudo apt-get update

And finally, install Certbot from the new repository with apt-get:

ajay@DataBucket:~$ sudo apt-get install python-certbot-apache

The certbot Let's Encrypt client is now ready to use.

Step 2 — Set Up the SSL Certificate

Generating the SSL certificate for Apache using Certbot is quite straightforward. The client will automatically obtain and install a new SSL certificate that is valid for the domains provided as parameters.

To execute the interactive installation and obtain a certificate that covers only a single domain, run the certbot command like so, where databucketsolutions.com is your domain:

ajay@Databucket:~$ sudo certbot --apache -d databucketsolutions.com

If you want to install a single certificate that is valid for multiple domains or subdomains, you can pass them as additional parameters to the command.

The first domain name in the list of parameters will be the base domain used by Let’s Encrypt to create the certificate, and for that reason we recommend that you pass the bare top-level domain name as first in the list, followed by any additional subdomains or aliases:

ajay@DataBucket:~$ sudo certbot --apache -d databucketsolutions.com -d www.databucketsolutions.com

For this example, the base domain will be databucketsolutions.com.

If you have multiple virtual hosts, you should run certbot once for each to generate a new certificate for each. You can distribute multiple domains and subdomains across your virtual hosts in any way.

You will be asked to provide an email address for lost key recovery and notices, and you will be able to choose between enabling both http and https access or forcing all requests to redirect to https.

When the installation is finished, you should be able to find the generated certificate files at /etc/letsencrypt/live. You can verify the status of your SSL certificate with the following link (don’t forget to replace databucketsolutions.com with your base domain):

ajay@DataBucket:~$ https://www.ssllabs.com/ssltest/analyze.html?d=databucketsolutions.com&latest

You should now be able to access your website using a https prefix.

Generating the second SSL certificate

ajay@DataBucket:~$ sudo certbot --apache -d dashboard.databucketsolutions.com -d www.dashboard.databucketsolutions.com

For this example, the base domain will be dashboard.databucketsolutions.com.

When the installation is finished, you should be able to find the generated certificate files at /etc/letsencrypt/live. You can verify the status of your SSL certificate with the following link (don’t forget to replace dashboard.databucketsolutions.com with your base domain):

ajay@DataBucket:~$ https://www.ssllabs.com/ssltest/analyze.html?d=dashboard.databucketsolutions.com&latest

You should now be able to access your website using a https prefix.

Step 3 — Verifying Certbot Auto-Renewal

Let’s Encrypt certificates only last for 90 days. However, the certbot package we installed takes care of this for us by running certbot renew twice a day via a systemd timer.

On non-systemd distributions this functionality is provided by a cron script placed in /etc/cron.d. The task runs twice daily and will renew any certificate that's within thirty days of expiration.

To test the renewal process, you can do a dry run with certbot:

ajay@DataBucket:~$ sudo certbot renew --dry-run

If you see no errors, you're all set.

When necessary, Certbot will renew your certificates and reload Apache to pick up the changes.

If the automated renewal process ever fails, Let’s Encrypt will send a message to the email you specified, warning you when your certificate is about to expire.

In this guide, we saw how to install a free SSL certificate from Let’s Encrypt in order to secure a website hosted with Apache. We recommend that you check the official Let’s Encrypt blog for important updates from time to time.
